# StartupGrow - Brief Project Exam Answer

Project: StartupGrow - Digital Solutions Platform for Indian Startups  
Team Members: Yadunand Pavithran, Nashith Nihal C, Gokul Krishna  
Live Website: https://startup-grow-sand.vercel.app/

## 1. About Your Project

StartupGrow is a web-based digital solutions platform created for early-stage Indian startups. Many startups have good business ideas but struggle with websites, branding, SEO, social media, content, PPC advertising, and email marketing because of limited budget, time, and technical knowledge. StartupGrow solves this problem by bringing these digital growth services into one simple platform.

The website allows founders to understand the services, view project details, submit a service request, and share their startup goals. The admin team can view and manage submitted requests through the SHIELD dashboard.

## 2. Synopsis

StartupGrow is designed as a one-stop platform for startup digital growth. It provides six core services: Search Engine Optimisation, Pay-Per-Click Advertising, Social Media Marketing, Content Marketing, Email Marketing, and Web Design and Development.

The system contains a public website, service request form, backend API, MongoDB database, and admin dashboard. A founder submits details such as startup name, founder name, city, startup stage, service required, budget, and project goal. The backend validates the request and stores it. The admin dashboard displays the request and allows status updates such as New, Contacted, In progress, and Closed.

## 3. Technology Stack

Frontend:
- Next.js for routing, rendering, and project structure.
- React for reusable components, forms, and dashboard views.
- HTML5, CSS3, JavaScript, Tailwind CSS, and Bootstrap/template CSS for UI.
- GSAP, ScrollTrigger, Swiper, and other scripts for animations and interactions.

Backend:
- Node.js and Express.js for backend logic.
- Next.js API routes for same-origin API support on Vercel.
- Mongoose for MongoDB schema modeling.
- MongoDB for storing service requests and contact leads.

Deployment and Tools:
- Vercel for hosting the website.
- GitHub for source code management.
- Visual Studio Code and Node.js for development.

## 4. System Design and Diagrams

System Architecture:
User Browser -> Next.js Frontend -> API Routes / Express API -> MongoDB Database -> Admin Dashboard

Main Modules:
- User Interface Module: Home, About, Services, Works, Team, FAQ, Blog, and Contact pages.
- Service Module: Displays SEO, PPC, Social Media Marketing, Content Marketing, Email Marketing, and Web Development services.
- Contact and Request Module: Collects founder and startup details through forms.
- Admin Module: SHIELD dashboard for viewing requests, leads, and updating statuses.

Module Diagram:
StartupGrow System -> User Interface Module, Service Module, Contact and Request Module, Admin Module.

## 5. Data Flow Diagram

DFD Level 0:
- External Entity: Startup Founder.
- Process: StartupGrow Platform.
- External Entity: Admin Team.
- Data Store: Service Request Data.
- Flow: Founder submits request to platform. Platform stores data and sends request details to the admin team. Admin updates status through the platform.

DFD Level 1:
- Founder explores services.
- Founder submits a service request.
- System validates required fields and selected service.
- Valid request is stored in MongoDB.
- Admin dashboard reads requests from the database.
- Admin updates request status.
- Updated status is stored and displayed again in the dashboard.

## 6. Entity-Relationship Diagram

Main Entities:
- ServiceRequest: Stores startup service inquiries.
- ContactLead: Stores general contact form leads.
- Service: Logical/static list of available services.
- Admin: Logical/future entity for dashboard users.

Relationships:
- One Service can be selected by many ServiceRequests.
- One Admin can manage many ServiceRequests.
- ContactLead is managed by the admin team.

## 7. Database Design - Tables with Attributes

Collection: ServiceRequest
- _id: Primary key generated by MongoDB.
- startupName: Name of the startup.
- founderName: Name of the founder.
- email: Founder email address.
- phone: Contact number.
- city: Startup city.
- stage: Startup stage, such as Idea stage, MVP launched, Early customers, or Scaling.
- service: Selected service.
- budget: Budget range.
- message: Project goal or requirement.
- status: New, Contacted, In progress, or Closed.
- createdAt: Request creation time.
- updatedAt: Last update time.

Collection: ContactLead
- _id: Primary key generated by MongoDB.
- name: Lead name.
- email: Lead email address.
- phone: Contact number.
- subject: Contact subject.
- service: Interested service.
- message: Lead message.
- status: New, Contacted, In progress, or Closed.
- createdAt: Lead creation time.
- updatedAt: Last update time.

Logical Collection: Service
- id: Service id.
- name: Service name.
- category: Service category.
- description: Short service description.

Logical/Future Collection: Admin
- id: Admin id.
- name: Admin name.
- email: Admin email.
- role: Admin role.

## 9. Conclusion

StartupGrow is a practical and useful platform for Indian startups. It combines website development and digital marketing services into one simple system. The project demonstrates frontend development, backend API handling, MongoDB database design, request management, and admin dashboard workflow.

The current version is suitable for academic demonstration and can be expanded into a stronger production platform with authentication, payments, analytics, client dashboards, and automation.
